About
Foreword
A living reference for the glyph site design system. When a pattern changes, update this document too.
Pages
index.html light marketing landing page — alternating light/dark zones start.html dark documentation — Get Started tutorial concepts.html dark documentation — Concepts reference api.html dark documentation — API reference with interactive search
Design Intent
Minimal, precise. Monospace body type signals developer audience without saying so. The accent red (--accent: #c44040) appears sparingly — active states and gradient borders only. When in doubt, leave it out.
Tone
Technical and direct. Code demonstrates; prose supports. Short sentences, no filler. Documentation pages assume the reader writes Go. Light zone marketing copy is confident but never hyperbolic.
Maintenance
tokens.css is the single source of truth for all design tokens — link it first on every page. Update this styleguide whenever a new pattern is established. A stale reference is worse than none.

05
Responsive
Breakpoints
Three breakpoints, defined consistently across all documentation pages.
@media (max-width: 640px) mobile — nav padding tightens to --s2, nav-links gap narrows to 10px @media (min-width: 960px) desktop — sidenav appears; .page + .nav-inner gain margin-left: 216px @media (min-width: 1492px) wide — .page + .nav-inner revert to margin: 0 auto
960px is the primary layout break. Below it, .sidenav hides and .topnav (horizontal scrolling tab strip) appears instead. The 1492px threshold prevents content drifting too far right on ultra-wide displays.

Consistency
Audit
Issues found across pages. Use this as a checklist when updating the site.
Fixed: Undefined CSS variable --berk — corrected to var(--mono) in index.html.
Fixed: Prose text color — .concept-prose now uses var(--fg3) across all pages.
Fixed: Comment syntax highlight color — all pages now use .code .cm { color: #444 }.
Fixed: Partial token sets — all pages now link tokens.css as single source of truth. Inline :root blocks removed.
Fixed: --akz-c removed from tokens.css (unused).
Fixed: Nav class names unified — start.html and concepts.html now use .nav / .nav-inner / .nav-brand / .nav-links matching index.html and api.html.
Fixed: Section label/title class names unified — .concept-label / .concept-title renamed to .section-label / .section-title on start.html and concepts.html.
Fixed: Section head bottom spacing — margin-bottom on start.html / concepts.html changed from var(--s3) to var(--s4), matching index.html and api.html.
Hardcoded dark-page text colours not tokenised
#c8c4b8, #e8e4d8, #888, #777, #666, #555, #444, #333, #1e1e1c are all used directly throughout dark-page styles. These are consistent in intent but not backed by tokens, making them fragile to a palette shift.
Spacing scale is consistent — all padding and gap values use --s1–--s8 tokens across all pages.
Max content width is consistent — 1060px applied via .zone-inner and .page on all pages.
Nav height is consistent — 48px across all nav variants.
Accent colours are consistent — all spinning border animations use the same conic gradient stops.
Font sizes and weights are consistent — body 13px / Berkeley Mono applies globally.
